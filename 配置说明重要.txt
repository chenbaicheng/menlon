1: 不要改动已经存在的文件位置 和名字 , 但是可以添加新的和删除你自己创建的文件 
2. 创建一个数据库 名字为 imoment  ,编码为utf-8
CREATE DATABASE imoment DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
eg: 如果要使用其他名字,请修改db.config.php ,保持config文件和创建的数据库名字一致  



*********************另外要点说明

1. 注意创建表的时候 全部会变成小写  所以在检查用户名的时候要将 用户名全部变成小写 然后再比较 此用户是否存在 在索引表中

2. 在注册初始化表时候  需要注意 有些外键无法实现,因为其他用户表还没有创建  所以需要交给业务层处理 ,使用php  检测 数据一致性

3. medoo 如果没有结果 ,会返回 array(0) { }  ,使用count() 检查 如果为0 就是为空

4.不建议在类的构造函数里面初始化其他类 ,最好使用形参传递的方法 ,好像viewModel 需要他自己的对象来处理数据库,这时候应该在类成员函数中传入,这个类的对象,而不是在构造函数中实例化的此类的对象,然后将类对象赋值给成员变量,  会出现很严重的程序错误,请不要这样做

5. 帖子的计数统计  由controller->ViewModel->ViewModel_Plug_in类负责
